# ModL10n
![image](example.png)

## 概要 (Description)
**ModL10n** は、Google Gemini APIを利用してMinecraftのModを半自動的に日本語へ翻訳するデスクトップアプリケーションです。
Modの `.jar` ファイルから英語の言語ファイル（`en_us.json`）をスキャンし、差分のみを抽出して翻訳を行うため、APIのコストを抑えつつ効率的に日本語化リソースパックを作成することができます。

## 主な特徴 (Features)
- **スマートな差分翻訳**: 過去に作成した翻訳データや、Mod内蔵の公式日本語訳を自動で読み込み、不足しているキーのみをGemini APIで翻訳します。
- **リソースパック自動生成**: 翻訳結果は指定したMinecraftのバージョンに合わせた `pack.mcmeta` と共に `output` フォルダに出力され、そのままリソースパックとして導入可能です。
- **用語集（Glossary）機能**: 固有のアイテム名や用語の翻訳ブレを防ぐため、設定ファイルから用語集を適用できます。
- **不要キーの自動整理**: Modのアップデートで使われなくなった古い翻訳キーを自動検出し、別ファイルに退避します。
- **組み込みコンソールコマンド**: アプリケーション内のコンソールから、翻訳データの検索、抽出、マージなどを対話的に実行できます。JSONファイルを直接編集する手間を省きます。

## 必須要件 (Requirements)
- **Python 3.8** 以上
- **Google Gemini API Key** (Google AI Studioから無料で取得可能)

### 必要なライブラリのインストール
本リポジトリには依存関係をまとめた `requirements.txt` が同封されています。
コマンドプロンプトまたはターミナルを開き、以下のコマンドを実行して必要なライブラリを一括インストールしてください。

```bash
# （推奨）仮想環境を作成して有効化する場合
python -m venv venv
# Windowsの場合
venv\Scripts\activate
# macOS/Linuxの場合
source venv/bin/activate

# パッケージのインストール
pip install -r requirements.txt
```
*(※最低限動作に必要な主なパッケージは `PyQt5` と `google-generativeai` です。)*

## 使い方 (Usage)

### 1. 準備
1. スクリプト (`ModL10n.py`) を実行します。
2. 初回起動時、自動的に `input` フォルダと `output` フォルダが作成されます。
3. 翻訳したいModの `.jar` ファイルを `input` フォルダに配置します。

### 2. アプリケーションの操作
1. **APIキーの設定**: 左メニューの「Gemini API Key」欄に取得したAPIキーを入力します（有効な場合、✔ Verified と表示されます）。
2. **バージョンの選択**: プレイするMinecraftのバージョンを選択します。
3. **対象Modの選択**: 「スキャン更新」ボタンを押すと、`input` フォルダ内のModがリストアップされます。翻訳したいModにチェックを入れます。
4. **翻訳実行**: 「翻訳実行」ボタンを押すとバックグラウンドで処理が開始されます。進行状況は右側のログエリアに表示されます。
5. **完了**: 処理が完了すると、`output` フォルダにリソースパック形式（`assets/` および `pack.mcmeta`）で出力されます。これをMinecraftの `resourcepacks` フォルダに入れて適用してください。

## コンソールコマンド (Console Commands)
画面右下の入力欄からコマンドを打ち込むことで、出力された翻訳データの管理が可能です。

| コマンド | 説明 |
| :--- | :--- |
| `/reload` | Modリストのスキャンを再実行します。 |
| `/start` | 翻訳処理を開始します（「翻訳実行」ボタンと同じ）。 |
| `/search value [ワード]` | 翻訳後の「値（日本語）」から特定のワードを含むエントリを検索します。 |
| `/search key [ワード]` | 翻訳前の「キー（英語）」から特定のワードを含むエントリを検索します。 |
| `/delete` | `/search` でヒットしたエントリを翻訳データから一括削除します。 |
| `/save` | `/search` でヒットしたエントリを別のJSONファイルとして保存します。 |
| `/extract` | `/search` でヒットしたエントリを別ファイルに保存し、元のデータからは削除します。 |
| `/merge [ファイル名]` | 指定した外部JSONファイルを読み込み、現在の翻訳データ（`output`内）の該当キーを上書きします。手動修正したデータの統合に便利です。 |

## 設定ファイル (Configuration)
初回起動後、スクリプトと同じ階層に `config.json` が生成されます。
このファイルを編集することで、プロンプトの調整や用語集の登録が可能です。

```json
{
    "minecraft_version": "1.21",
    "system_prompt": "以下のJSON形式のMinecraft Modの言語データを日本語に翻訳してください...",
    "glossary": {
        "Iron Ingot": "鉄インゴット",
        "Wand": "魔法の杖"
    }
}
```
- **glossary**: ここに英単語と指定したい日本語訳を「キー: 値」の形式で追加することで、AIの翻訳精度と統一性を向上させることができます。

## ディレクトリ構成
```text
📦ModL10n
 ┣ 📜ModL10n.py          # メインプログラム
 ┣ 📜requirements.txt    # 必要なパッケージ一覧
 ┣ 📜config.json         # 設定ファイル（用語集・プロンプト）
 ┣ 📜.env                # APIキー保存用（自動生成）
 ┣ 📜deleted_entry.json  # 削除された古い翻訳キーの履歴（自動生成）
 ┣ 📂input               # 翻訳したいMod(.jar)を入れるフォルダ
 ┗ 📂output              # 翻訳結果が出力されるフォルダ（リソースパックとして機能）
```
